# 全量达人数据补充、字段导出与模板保存交付说明

## 本轮改造概述

本轮我已将你上传的**全量达人数据接口脚本**改写并接入到 `kol_lens` 的真实产品流程中，使其能够在工作台里承接“**选中达人后，一键补充全量数据**”这一动作，并进一步支持“**按字段选择导出**”与“**保存导出模板供后续复用**”两项能力。

这次改造不是简单地把原脚本搬进仓库，而是将其中可复用的接口调用、字段解析与数据清洗逻辑重构为独立后端服务，并去除了脚本式、强耦合和高风险的实现方式，使其适配当前项目已有的资产库、检索结果和工作台交互结构。

## 新增/扩展接口

| 接口 | 方法 | 作用 |
|---|---|---|
| `/api/v1/creator-data/catalog` | `GET` | 返回可补充/可导出的字段目录、字段分组与默认字段集合 |
| `/api/v1/creator-data/enrich` | `POST` | 对当前选中达人执行全量数据补充，返回结构化表格行数据 |
| `/api/v1/export/templates` | `GET` | 查询当前品牌、SPU、操作人维度下的导出模板列表 |
| `/api/v1/export/templates` | `POST` | 保存或更新导出模板 |
| `/api/v1/export/creators` | `POST` | 按所选字段导出当前达人数据，生成可下载文件 |
| `/api/v1/export/download/{file_name}` | `GET` | 下载生成好的 CSV 导出文件 |

## 工作台交互变化

在前端工作台中，当前流程已经扩展为如下形态：用户完成检索并选中达人后，可以直接在结果区上方看到一块新的**Creator Data Booster** 面板。该面板包含三层能力。

第一层是**一键补充数据**。点击后，前端会把当前选中的达人列表与勾选字段传给后端，由后端调用服务化后的全量数据接口，并将解析好的字段结果回填成表格。

第二层是**字段级导出控制**。左侧按照字段分组展示所有可选维度，用户可以勾选需要展示和导出的字段；右侧则实时以表格方式展示这些字段在当前选中达人上的补充结果。

第三层是**导出模板复用**。用户可以选择已有模板、保存当前字段组合为模板，并在下一次使用相同品牌/SPU时直接套用，从而避免每次重复勾选导出字段。

## 对上传脚本的优化方式

上传脚本中的原始逻辑已经被拆分并优化为更适合线上服务的结构。核心优化点如下。

| 优化方向 | 本次处理 |
|---|---|
| 脚本式主流程 | 改造成 `CreatorDataService` 服务对象，供 API 直接调用 |
| 字段映射分散 | 收敛为统一字段目录 `FIELD_CATALOG`，前后端共用同一套字段 key/label 逻辑 |
| 原始接口返回不稳定 | 增加安全解析、类型归一、列表规范化和兜底格式化逻辑 |
| 模板无持久化能力 | 新增 JSON 文件模板存储层 `ExportTemplateStore` |
| 导出逻辑临时化 | 增加标准 CSV 导出与下载路径解析能力 |
| 高风险实现 | 去除不适合直接落入产品仓库的脚本式 cookie/文件处理方式，保留可控服务化封装 |

## 当前实现边界说明

当前版本已经满足你提出的主流程目标，即：**选择完博主后，可以点击一键补充数据，看到包含全量数据维度的表格，并在导出时自由选择字段，同时保存模板供后续复用**。

不过，这一版仍有两个后续可以继续增强的方向。其一是把补充出来的字段进一步沉淀回 `library/history` 或达人资产详情页，形成跨批次可复用的“数据快照缓存”；其二是把模板能力从单纯字段模板继续扩展为“字段 + 排序 + 过滤条件 + 品牌/SPU 默认视图”的完整导出方案模板。

# Σ.Match (易美寻星) 智能工作站前台产品需求与交互规范 (PRD & UI/UX Spec)

## 一、 产品概述与全局基建

Σ.Match 是一款集“自然语言检索、多维数据透视、沉浸式达人评审、资产自动化沉淀与履约履历管理”于一体的内部 SaaS 系统。本规范定义了前端页面的组件树、数据流转逻辑及核心交互，并结合了暗黑科技（Cinematic Dark Tech）的视觉设计语言。

### 1.1 全局状态管理 (Store)

前端需维护以下全局核心状态，确保跨页面、跨组件的数据一致性与算法连续性：

*   **`ProjectContext` (项目上下文)**：贯穿全工作流。包含 `Brand_Name` (品牌名)、`SPU_Name` (产品名)。
*   **`UserRole` (角色鉴权与权重)**：定义当前操作人身份。影响反向打标时的权重分配。层级定义：客户 (最高权重) > 前端/策划 (中等权重) > 采购 (基础权重)。
*   **`IntentState` (意图状态)**：记录当前解析出的筛选项及各维度的“宽容度权重”。
*   **`FeedbackPool` (决策池)**：跨组件维护三个数组：`Selected_Influencers` (选中)、`Pending_Influencers` (待定)、`Rejected_Influencers` (淘汰)。

### 1.2 视觉与设计系统规范 (Design System)

参考现有的暗黑科技美学，整体 UI 采用以下规范：

*   **色彩体系**：
    *   背景色：纯黑底色 `#080808`，配合玻璃拟态 (Glassmorphism) 模糊效果。
    *   品牌主色：易美红 `#D4001A`，用于强调、按钮悬浮、发光特效 (Glow effects)。
    *   面板/卡片色：深灰 `#110000` 或 `#140000`，带有微弱的红色边框发光。
*   **排版字体**：
    *   展示字体 (Display)：Bebas Neue 或 Space Grotesk。
    *   中文字体：Noto Sans SC。
    *   数字/数据字体：Orbitron (用于展示各项数据指标，增强科技感)。
*   **动效与微交互**：
    *   扫描线 (Scanlines) 与噪点 (Noise) 叠加层，营造终端控制台氛围。
    *   卡片悬浮时带有红色呼吸灯效果 (`box-shadow: 0 0 30px rgba(212, 0, 26, 0.2)`)。

## 二、 核心页面与交互规范

### 2.1 导航与入口页 (`/home`)

*   **UI 呈现**：全屏工作台引导页，包含产品介绍、使用规范。背景可采用粒子动画或深色科技感背景。
*   **核心交互**：点击“新建寻星任务”按钮，强制唤起 `<ContextInitModal>` 弹窗。
*   **控件包含**：
    *   品牌名下拉/输入组合框 (Combobox)。
    *   SPU 名下拉/输入组合框。
    *   **角色身份展示**：显性展示当前登录身份角色（采购/策划/客户）。
*   **校验逻辑**：必填项，未填写或未选择无法进入工作台。填写完毕后，将数据写入全局 `ProjectContext`，路由跳转至 `/workspace`。

### 2.2 智能检索工作台 (`/workspace`)

该页面为核心功能区，采用左侧对话流、右侧悬浮面板的布局。

#### 2.2.1 需求录入与意图解析 (`<ChatInterface>`)

*   **交互流转**：
    1.  用户在 `<ChatInput>` (支持富文本与快捷回车的文本域) 输入自然语言需求并发送。输入框附近需增加设置项 `[期望寻回数量：默认 10 ∨]`。
    2.  渲染系统回复气泡，呈现 `<IntentDashboard>` 组件。
*   **`<IntentDashboard>` 组件规范**：
    *   **功能**：展示系统解析出的结构化条件，并允许用户干预。
    *   **UI 呈现**：分为“基础筛选项”、“视觉/标签项”、“SPU 推荐特征”三个区块。
    *   **控件规范**：每一个条件右侧必须绑定一个 `<Slider>` (滑块) 组件，用于调节“权重/宽容度”（1-5 档）。
    *   **标签视觉区分**：
        *   **SPU 公有标签**：背景采用品牌色，带图标（如 🌟），Hover 提示：“该 SPU 历史高频命中特征”。
        *   **用户私有标签**：背景采用中性色，带图标（如 👤），Hover 提示：“基于您的历史筛选习惯自动附加”。

#### 2.2.2 弹性寻回加载中枢 (`<ElasticLoadingTerminal>`)

*   **功能**：缓解查询焦虑，展示黑盒的降级过程。
*   **UI 表现**：采用类似终端（Terminal）打字机的效果，实时渲染后端返回的执行日志。例如：`"本地库仅命中 4 人，触发全网寻回，目标补齐至 10 人... 已寻回 7/10... 10/10 寻回完毕。"`

#### 2.2.3 超宽数据矩阵 (`<DataGridList>`)

*   **触发条件**：系统完成寻回后，在对话流中渲染出达人列表。
*   **交互规范**：
    *   启用**水平滚动 (Horizontal Scroll)**。
    *   **首列冻结 (Sticky Column)**：最左侧的“博主信息”与“近期笔记”列必须固定。
    *   **系统级核心字段**：紧贴冻结列之后，强插入两列不可隐藏的字段：`综合匹配度 (Match Score)` 和 `性价比指数 (ROI Proxy)`。
    *   **异步加载 (Async Loading)**：对于需要额外调接口的字段，渲染为闪烁的占位符 (Skeleton)，数据返回后静默更新。
    *   **视图分组控件**：提供 `<Tabs>` 控件切换列组。
*   **数据列定义 (共 34 维)**：
    *   **[基础视图]**：博主信息、近期笔记、粉丝数、粉丝量变化幅度、活跃粉丝占比、互动粉丝占比。
    *   **[日常大盘]**：曝光中位数(日常)、阅读中位数(日常)、互动中位数(日常)、千赞笔记比例、百赞笔记比例。
    *   **[图文表现]**：图文曝光中位数(日常)、图文阅读中位数(日常)、图文互动中位数(日常)、图文千赞笔记比例、图文百赞笔记比例。
    *   **[视频表现]**：视频曝光中位数(日常)、视频阅读中位数(日常)、视频互动中位数(日常)、视频千赞笔记比例、视频百赞笔记比例、视频完播率。
    *   **[合作转化]**：曝光中位数(合作)、阅读中位数(合作)、互动中位数(合作)、外溢进店中位数、外溢进店单价。
    *   **[预估报价]**：图文预估 CPM 价格、图文预估阅读单价、图文预估互动单价、视频预估 CPM 价格、视频预估阅读单价、视频预估互动单价、全部报价。

#### 2.2.4 沉浸式评审弹窗 (`<ReviewModal>`)

*   **触发条件**：点击 `<DataGridList>` 中的任意一行。
*   **UI 呈现**：全屏或 90% 宽度的遮罩弹窗。
    *   **左侧侧边栏**：基础信息概览、全量报价。
    *   **右侧顶栏**：数据可视化雷达图。
    *   **右侧主体**：类似小红书主页的近 20 篇笔记双列瀑布流展示。
*   **核心动作与连续翻牌交互**：
    *   底部固定悬浮操作栏，包含：【选中 (Target)】、【待定 (Pending)】、【未选中 (Reject)】。
    *   点击任意按钮后，**弹窗不关闭**，触发向左滑动的过渡动画，自动渲染下一位达人数据。
    *   遍历完最后一位达人后，弹窗自动关闭，原表格中对应的行前渲染对应的状态 Icon。

#### 2.2.5 进化决策坞 (`<FissionDock>`)

*   **UI 呈现**：常驻页面底部的吸底面板。显示已选中达人的头像缩略图横排列表。
*   **动作 1：【获取更多 (Fission)】**
    *   点击后触发整页 Loading。将当前选中的达人特征作为反向检索参数，发起新一轮请求。
    *   加载完成后，渲染新一批 `<DataGridList>` 和更新后的 `<IntentDashboard>`，并通过提示条告知用户系统自动调整了哪些权重。
*   **动作 2：【确认入库 (Commit)】**
    *   结束当前寻星流。将已选中的达人结合当前的 `ProjectContext` 提交后端执行多维打标。
    *   **权限差异化**：若当前登录为“客户”，按钮应带有发光特效，提示“您的选择将作为该 SPU 核心基准向量”。

### 2.3 达人资产库与履约中心 (`/library`)

*   **UI 呈现**：标准的上下分栏后台管理页布局。
*   **顶部筛选区 (`<FilterBar>`)**：
    *   品牌/SPU 联动下拉框。
    *   入库时间范围、入库人角色筛选。
*   **数据区 (`<LibraryTable>`)**：支持跨页多选的达人数据表。
*   **侧边栏：履约历史抽屉 (`<HistoryDrawer>`)**：
    *   通过时间轴组件渲染历史操作记录。
    *   **深度下钻**：时间轴节点为可点击卡片。点击邀约记录可查看文案快照和达人列表；点击下单记录可查看表单快照。
*   **底部动作坞 (`<BatchActionPanel>`)**：
    *   **【批量邀约】**：弹窗展示富文本编辑器，生成邀约文本。
    *   **【批量下单】**：弹窗展示对接官方平台合作标准的动态表单。

---

### 2.4 智能导出与字典沉淀引擎 (`<SmartExportModal>`)

*   **触发条件**：在资产库点击【导出 Excel/CSV】按钮。
*   **交互流转**：
    *   **Step 1：表头录入**。提供文本域粘贴表头，或上传示例空表解析首行。
    *   **Step 2：精准比对匹配**。与后端标准映射字典完全匹配的词汇，渲染为绿色标签“已自动匹配”。
    *   **Step 3：AI 语义排异与用户确认**。未匹配词汇渲染为待办列表，由后端大模型提供候选选项（含“忽略此列”）。用户必须逐一勾选确认。
    *   **Step 4：导出与知识库沉淀**。生成文件下载，同时将用户手动确认的非标准词汇映射对持久化至系统字典。

## 三、 核心 API 数据交互契约 (Payload 定义)

### 3.1 意图解析请求 (`POST /api/v1/intent/parse`)
```json
// Req
{ "raw_text": "找几个上海的高冷风女博主" }

// Res
{
  "code": 200,
  "data": {
    "hard_filters": { "location": ["上海"], "gender": "女" },
    "soft_tags": ["高冷风"],
    "weights": { "location": 5, "gender": 5, "soft_tags": 3 }
  }
}
```

### 3.2 弹性检索与反馈进化请求 (`POST /api/v1/match/retrieve`)
```json
// Req
{
  "parsed_intent": { /* API 1 确认后的完整树 */ },
  "feedback": {
    "selected_ids": [1001, 1045],
    "rejected_ids": [1088, 1092]
  },
  "pagination": { "limit": 10 }
}
```

### 3.3 资产入库与打标 (`POST /api/v1/assets/commit`)
```json
// Req
{
  "project_context": {
    "brand_name": "某奶粉品牌",
    "spu_name": "高端系列 A 段",
    "operator_role_id": 2 // 1:采购, 2:策划, 3:客户
  },
  "influencers": [
    { "id": "red_10023", "selected_reason_tags": ["互动率高", "高冷风"] }
  ]
}
```

### 3.4 智能表头匹配与字典沉淀 (`POST /api/v1/export/confirm-mapping`)
```json
// Req
{
  "user_confirmed_mappings": [
    {
      "user_input_header": "视频均赞",
      "mapped_standard_key": "视频互动中位数(日常)"
    }
  ],
  "ignored_headers": ["备注说明", "内部评级"]
}
```
